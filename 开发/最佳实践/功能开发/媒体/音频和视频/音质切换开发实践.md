# 音质切换开发实践

更新时间：2026-04-01 09:49:00

来源：https://developer.huawei.com/consumer/cn/doc/best-practices/bpta-sound-quality-switching

#### 概述

在音乐播放应用中，同一首歌曲在不同音质间的切换是一种常见功能，而音频文件的压缩方式是影响音质的关键因素之一。下表展示了不同压缩方式对应的音质对比：
  
| 类型 | 代表格式 | 播放能力支持 | 特点 |
| --- | --- | --- | --- |
| 未压缩 | PCM/WAV | PCM：AudioRenderer WAV：AVPlayer | 高保真、体积大 |
| 无损压缩 | FLAC/APE | AVPlayer | 可还原原始数据 |
| 有损压缩 | MP3/AAC | AVPlayer | 体积小、音质损失 |
 
 
音质可以在未压缩、无损压缩和有损压缩之间切换，根据上述播放能力支持范围可以看出，音质切换的本质为[AudioRenderer](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-audio-audiorenderer)与[AVPlayer](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-media-avplayer)的播放切换，以及AVPlayer与AVPlayer的播放切换（即使用AVPlayer在不同音频格式间的播放切换）。
 

 
本文将介绍以下两种音质切换场景的实现：
 
- [AudioRenderer与AVPlayer播放切换](#section1718774317213)
- [AVPlayer与AVPlayer播放切换](#section198441182235)

 
 

#### AudioRenderer与AVPlayer播放切换

 

#### 场景描述

本场景以同一音频文件的PCM格式与MP3格式之间进行切换播放为例，切换后保持音频播放进度一致，即无缝衔接。
 

![](assets/音质切换开发实践/file-20260515114749588-0.gif)

 
 

#### 实现原理
1. 使用AudioRenderer播放PCM文件，即MP3音质（标清）切换到无损未压缩的PCM音质时，通过跳转的时间戳计算出新的offset偏移位置，待下次读取音频数据时，从该offset偏移位置开始读取，AudioRenderer即可实现跳转至指定位置播放。
2. 使用AVPlayer播放MP3文件，即无损未压缩的PCM音质切换到MP3音质（标清）时，通过AVPlayer的[seek()](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-media-avplayer#seek9)接口可直接跳转到对应时间戳位置播放。
 
 

#### 开发步骤
1. 在MediaTools类中，定义如下方法：
- msToCountdownTime()：根据当前毫秒数转换为分钟格式。

2. getMsFromByteLength()：根据字节长度转化为播放时间长度。

3. getOffsetFromTime()：根据播放时间长度转化为字节长度。

4. 在AVPlayerController类中，定义如下方法：
reset()：在其中调用AVPlayer实例的[reset()](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-media-avplayer#reset9)方法，重置播放。

5. pause()：在其中调用AVPlayer实例的[pause()](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-media-avplayer#pause9-1)方法，暂停播放。

6. seek()：在其中调用AVPlayer实例的[seek()](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-media-avplayer#seek9)，跳转播放。

7. 在AudioRendererController类中，定义如下方法：
initAudioRenderer()：初始化，获取AudioRenderer实例。

8. setWriteDataCallback()：写入音频数据。

9. start()：在其中调用AudioRenderer的[start()](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-audio-audiorenderer#start8)方法，启动音频渲染器。

10. pause()：在其中调用AudioRenderer的[pause()](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-audio-audiorenderer#pause8)方法，暂停音频渲染。

11. seek()：自定义seek()方法保存当前播放进度、时间的转换和字节长度，用于AudioRenderer跳转播放。

12. 在PlayerController类中，定义如下方法：
changeType()：传入[RawFileDescriptor](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-resource-manager#rawfiledescriptor9)音频资源描述符，并通过audioType的值，判断使用AudioRenderer或AVPlayer播放音频。

13. seek()：通过判断isAVPlayer()返回值，调用AVPlayerController或AudioRendererController对应的seek()方法，用于跳转指定位置播放音频资源。

14. isAVPlayer()：用于判断是否使用AVPlayer播放音频资源。

15. 在页面中自定义changeType()事件，通过[getRawfd()](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-resource-manager#getrawfd9-1)获取resources/rawfile目录下的音频资源，将获取到的数据传入PlayerController的changeType()方法，调用playerController的seek()方法传入当前Slider进度的值，用于跳转播放。
```ArkTS
private changeType(songSrc: string, audioType: AudioType): void {
  let current: number = this.value;
  this.getUIContext().getHostContext()?.resourceManager.getRawFd(songSrc).then((rawFileDescriptor) => {
    this.playerController.changeType(rawFileDescriptor, audioType);
    this.playerController.seek(current);
  }).catch((error: BusinessError) => {
    Logger.error(`resourceManager error code ${error.code} message ${error.message}`);
  });
}
```
 通过给对应MenuItem子组件添加对应点击事件，在其中调用changeType()方法传入音频资源文件路径和对应音频格式audioType，实现音频资源PCM格式与MP3格式的切换播放，即AudioRenderer与AVPlayer切换播放。

  
```ArkTS
Menu() {
  MenuItem({
    content: $r('app.string.standard_quality'),

  // ...

    content: $r('app.string.pcm_high_quality'),
  })
    .onClick(() => {
      if (this.index === 2) {
        return;
      }
      this.text = $r('app.string.pcm_high_quality');
      this.index = 2;
      this.changeType(this.songData.src + AudioName.PCM, AudioType.PCM);
      this.isPlay = true;
    })
}
// ...
```


  

  #### AVPlayer与AVPlayer播放切换

  

  #### 场景描述

  本场景以同一音频文件的FLAC格式与MP3格式之间进行切换播放为例，切换后保持音频播放进度一致，即无缝衔接。

  
![](assets/音质切换开发实践/file-20260515114749588-1.gif)


  

  #### 实现原理

  使用同一个AVPlayer实例播放FLAC文件和MP3文件，每次切换后对AVPlayer进行重置，重新设置音频资源url，然后通过AVPlayer的seek()接口直接跳转到对应时间戳位置播放。

  

  #### 开发步骤

1. 在MenuItem组件的点击事件中调用自定义changeType()方法，传入音频资源文件路径和对应音频格式audioType。
```ArkTS
Menu() {
  MenuItem({
    content: $r('app.string.standard_quality'),
  }).onClick(() => {
    if (this.index === 0) {
      return;
    }
    this.text = $r('app.string.standard_quality');
    this.index = 0;
    this.changeType(this.songData.src + AudioName.MP3, AudioType.MP3);
    this.isPlay = true;
  })
  MenuItem({
    content: $r('app.string.flac_high_quality'),
  })
    .onClick(() => {
      if (this.index === 1) {
        return;
      }
      this.text = $r('app.string.flac_high_quality');
      this.index = 1;
      this.changeType(this.songData.src + AudioName.FLAC, AudioType.FLAC);
      this.isPlay = true;
    })
  // ...
}
// ...
```


2. 同样的，在页面自定义changeType()事件中，通过getRawfd()获取音频资源，将获取到的数据传入PlayerController的changeType()方法，调用playerController的seek()方法传入当前Slider进度值，用于跳转播放。
```ArkTS
private changeType(songSrc: string, audioType: AudioType): void {
  let current: number = this.value;
  this.getUIContext().getHostContext()?.resourceManager.getRawFd(songSrc).then((rawFileDescriptor) => {
    this.playerController.changeType(rawFileDescriptor, audioType);
    this.playerController.seek(current);
  }).catch((error: BusinessError) => {
    Logger.error(`resourceManager error code ${error.code} message ${error.message}`);
  });
}
```


3. 在PlayerController的changeType()方法中：
通过传入的audioType的值，判断音频是否为MP3或FLAC格式。

4. 调用audioRendererController的pause()方法，暂停AudioRenderer音频渲染。

5. 通过avPlayerController的reset()方法重置音频资源的描述符、偏移量和文件长度，实现音频资源FLAC格式与MP3格式的切换播放，即AVPlayer与AVPlayer切换播放。

  
```ArkTS
export class PlayerController {
  // ...
  // Modifying the audio format
  changeType(rawFileDescriptor: resourceManager.RawFileDescriptor, audioType: AudioType) {
    try {
      this.rawFileDescriptor = rawFileDescriptor;
      if (audioType === 1 || audioType === 2) {
        this.audioRendererController?.pause(); // Pause AudioRenderer playback
        // Reset AVPlayer playback
        this.avPlayerController?.reset(this.rawFileDescriptor.fd, this.rawFileDescriptor.offset,
          this.rawFileDescriptor.length);
      } else {
        // ...
      }
      this.currentType = audioType;
    } catch (error) {
      Logger.error('changeType error!');
    }
  }
  // ...
  // Determine the return value of isAVPlayer() and call the corresponding seek() method
  seek(currentTime: number): void {
    if (this.isAVPlayer()) {
      this.avPlayerController?.seek(currentTime);
    } else {
      // ...
    }
  }
  // ...
}
```
