## DevEco Studio 5.0.0 Release（5.0.3.906）新增和增强特性


### 新增特性


- ohpm-repo新增use_reverse_proxy字段用于配置是否使用反向代理选项。具体请参考[配置项说明](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/ide-ohpm-repo-configuration-V5#zh-cn_topic_0000001745376470_配置项说明)。
- DevEco Studio提供Generate ArkTSDoc功能，将代码文件中变量、方法、接口、类等需要对外暴露的信息快速生成相应的参考文档。具体请参考[生成ArkTSDoc文档](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/ide-arktsdoc-V5)。
- hvigor-config.json5文件新增ohos.nativeResolver字段，用于指定ArkTS编译过程中是否使用高性能插件进行依赖寻址，使用高性能插件可以降低编译过程的峰值内存，加快编译速度。具体可参考[hvigor-config.json5](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/ide-hvigor-set-options-V5)。
- hvigor新增获取远程hsp路径的API：getOhpmRemoteHspDependencyInfo。具体请参考[插件上下文](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/ide-build-expanding-context-V5)。
- 应用与服务体检新增1条快速性能检测规则和2条最佳实践规则。具体请参考[评分方法及规则](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/ide-app-analyzer-V5#section7155192511196)。
- 模拟器支持selinux。


- DevEco Studio提供开箱即用的开发体验，将HarmonyOS SDK、Node.js、Hvigor、OHPM、模拟器平台等进行合一打包，简化DevEco Studio安装配置流程；并提供一体化的历史工程迁移能力，帮助开发者快速完成工程转换。具体请参考[一体化工程迁移](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/ide-integrated-project-migration-V5)。
- 使用ArkTS语言开发时，代码编辑新增支持Refactor-Convert能力。支持将字面量类型转换为映射类型等高频转换操作，辅助开发者高效重构代码。具体请参考[代码重构](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/ide-code-refactoring-V5)。
- 端云一体化工程支持创建、开发、调试、部署、调用云对象。具体请参考[开发云对象](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/agc-harmonyos-clouddev-cloudobj-V5)。
- 编译构建支持通过Build Analyzer工具可视化分析排查构建过程中的性能问题。具体请参考[分析构建性能](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/ide-hvigor-build-analyzer-V5)。
- Stage模型的ArkTS工程支持代码混淆。具体请参考[代码混淆](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/ide-build-obfuscation-V5)。
- 支持开发者自定义Hvigor任务和插件。具体请参考[扩展构建](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/ide-build-expanding-V5)。
- 提供hvigor生命周期的hook，便于开发者使用hook在生命周期中按需进行逻辑处理。具体可供开发者使用的hook请参考[构建生命周期](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/ide-hvigor-life-cycle-V5)。
- 新增运行时获取编译构建参数的功能。具体请参考[获取自定义编译参数](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/ide-hvigor-get-build-profile-para-V5)。
- 支持使用ArkUI Inspector分析布局，开发者可以在DevEco Studio上查看应用在真机上的UI显示效果。具体请参考[布局分析](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/ide-arkui-inspector-V5)。
- 支持使用Device File Explorer访问设备文件，开发者可以在DevEco Studio上对设备文件进行上传、下载等操作。具体请参考使用[查看设备文件](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/ide-device-file-explorer-V5)。
- 故障分析新增TSan检测和方舟多线程检测。具体可参考[TSan检测](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/ide-tsan-V5)和[方舟多线程检测](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/ide-multi-thread-check-V5)。
- 新增hstack工具，可用于将release应用混淆后的crash堆栈还原为源码对应堆栈。具体可参考[hstack](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/ide-command-line-hstack-V5)。

- 预览支持轻量化模拟，开发者能够使用预览器进行调试。具体请参考[调试概述](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/ide-debug-device-V5)。

### 变更特性


工程级build-profile.json5文件的compatibleSdkVersionStage字段新增枚举值：beta5、beta6、release。具体可参考build-profile.json5。

- Refactor-Convert代码转换功能操作由点击编辑器界面灯泡图标在下拉菜单中选择转换类型，变更为右键单击Refactor > Convert选择转换的方式。具体请参考[Refactor-Convert代码转换](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/ide-code-refactoring-V5#section760818259244)。
- Code Linter部分规则所属规则集或告警等级变更；@performance/hp-arkui-wrap-waterflow-if-else-footer规则下线。具体请参考[规则变更说明](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/ide-codelinter-rules-change-V5)。
- @security/specified-interface-call-chain-check新增对命名空间namespace、类型别名type、接口interface、枚举enum和结构体struct的支持。具体请参考[@security/specified-interface-call-chain-check](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/ide-specified-interface-call-chain-check-V5)。
- 已上架应用市场的应用不支持录制ArkTS Callstack、Callstack、Native Allocation、ArkTS Allocation、ArkTS Snapshot、ArkUI Component、ArkUI State泳道。
- HAR包的打包流程新增ProcessHarArtifacts任务。具体请参考[HAR基础任务流程图](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/ide-hvigor-task-process-V5#section988883610112)。
- DevEco Studio信息中心下线常见问题FAQ功能。如需查看常见问题解决方案，具体请参考[DevEco Studio FAQ](https://developer.huawei.com/consumer/cn/doc/harmonyos-faqs-V5/faqs-deveco-studio-V5)。


- API 11 Stage模型及以上，创建元服务工程或在元服务工程中创建模块时，不再默认创建服务卡片和EntryCard。开发者创建卡片时可任意选择卡片尺寸。
- 新建工程界面以及模板市场下架About Ability、Account、Business Card Ability、Category Ability、Category List Ability、Grid Ability、List Ability、Login Ability、Map Ability、Splash Screen Ability十个工程模板。
- API 12模板依赖的云开发SDK变更为Cloud Foundation Kit，开发者无需再显式引入云开发SDK作为依赖。具体请参考[端云一体化开发](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/agc-harmonyos-clouddev-view-V5)。
