#### 接入规范

为维护华为通知生态秩序，保障用户合法权益和良好的使用体验，根据现行法律法规及[《华为开发者服务协议》](https://developer.huawei.com/consumer/cn/doc/start/agreement-0000001052728169)、[《华为APIs使用协议》](https://developer.huawei.com/consumer/cn/doc/20209)、[《应用审核指南》](https://developer.huawei.com/consumer/cn/doc/50104)、[《元服务审核指南》](https://developer.huawei.com/consumer/cn/doc/app/50129)，特制定本规范。所有发布上架到华为应用市场且使用后台任务的HarmonyOS应用、联运应用及元服务（以下简称应用）应当遵守本规范。
 
  

#### 长时任务基本原则
 
| 总体原则 | 说明 |
| --- | --- |
| 不得在超出长时任务类型规定的场景使用，保证应用后台执行的业务与申请的任务类型一致。 | 特殊场景长时任务后台运行开关需由用户授权允许，应用使用特殊场景长时任务时需弹窗询问用户是否允许应用后台执行。 |
| 应用执行完长时任务业务后，应及时取消长时任务。 | - |
| 不得在非用户主动行为触发的场景，申请长时任务后台运行。 | - |
| 不得在用户主动停止（通过删除长时任务通知、删除AVSession通知等方式）并取消长时任务后，立即再次申请长时任务。 | - |
| 不得在系统检测到应用不合理使用长时任务，并取消/暂停长时任务后，立即再次申请长时任务。 | - |
| 不得修改长时任务通知内容。 | 使用数据传输任务时，应用需要更新进度。更新后的内容必须用于此场景，不能用作其他用途。 |
| 应用不得通过后台任务恶意“保活”。 | 恶意“保活”：指的是未经过用户允许和非正常业务需要的场景下，应用持续在后台运行。 |
| 避免出现其他可能影响终端用户体验的行为。 | - |
| 满足基本的性能、功耗要求。 | - |
 
 
  

#### 违规处罚标准

  

#### 违规分类、违规行为及违规处罚标准

当您的应用存在违规行为时，Background Task Kit会向应用的开发者账号发送违规通知邮件或短信，请您按照邮件或者短信中提供的步骤查看违规处罚详情。
  
| 违规分类及违规行为 | 违规处罚标准 |
| --- | --- |
| 违反长时任务指定的使用场景，影响用户体验。 违规的使用场景包括： · 应用在长时任务类型规定的场景范围外使用。 · 应用执行完规定场景的业务后，未及时取消长时任务。 · 在非用户主动行为触发的场景下，申请长时任务后台运行，用户对应用后台运行无明确的预期。 · 用户主动停止（通过删除长时任务通知、删除AVSession通知等方式）并取消长时任务后，立即再次申请长时任务。 · 在系统检测到应用不合理使用长时任务，并取消/暂停长时任务后，不得立即再次申请长时任务。 · 修改长时任务通知内容。 · 应用通过后台任务恶意“保活”。恶意“保活”，指的是未经过用户允许和非正常业务需要的场景下，应用持续在后台运行。 · 其他影响华为终端用户体验的行为。 | 根据违规产生的负面影响程度，采取对应处罚： · 严重影响：立刻在违规场景禁止使用长时任务。应用需在收到违规处罚通知后完成整改，并反馈整改结果。 · 轻微影响：应用需在收到违规处罚通知的7个自然日内完成整改，并反馈整改结果，否则在违规场景将禁止使用长时任务。 说明： 应用提交整改反馈时，请参考整改承诺声明。 |
