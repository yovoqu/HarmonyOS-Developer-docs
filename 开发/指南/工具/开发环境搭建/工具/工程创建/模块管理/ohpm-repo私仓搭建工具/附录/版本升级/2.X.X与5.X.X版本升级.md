# 2.X.X/5.X.X升级至更高版本

更新时间：2026-05-14 10:06:01

来源：https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/ide-upgrade-2xx_to_2xx

如需将ohpm-repo版本2.X.X/5.X.X版本升级到更高版本，可参考此文档。
 

![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/59/v3/ZsjRC0UkTrWsCHhCUqvHTA/notice_3.0-zh-cn.png?HW-CC-KV=V1&HW-CC-Date=20260528T015017Z&HW-CC-Expire=86400&HW-CC-Sign=DE7DDB48FF8E9522679244DB2D32D995EE496CB563743FCFC29F404225A52F59)
 

1. 在升级之前，请务必备份好ohpm-repo私仓工具中的历史数据，避免因升级操作失误，导致数据丢失。备份的内容包括[&lt;deploy_root&gt;](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/ide-upgrade-110_to_2xx#li10435216234)ohpm-repo部署根目录内数据，db元数据以及store三方包数据，详细可参考[数据备份](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/ide-ohpm-repo-data-backup) 。
 
2. 如果您想要改变db元数据和store三方包的存储方式，可在正确升级后参考[数据迁移文档指导](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/ide-ohpm-repo-data-migration)修改。
 

1. 旧版本服务停止：如果旧版本的服务还在运行，升级版本前请停止，执行stop。

  
```text
ohpm-repo stop
```


  
![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/c4/v3/2Ro1swe_RJGej4TV-BrWyA/caution_3.0-zh-cn.png?HW-CC-KV=V1&HW-CC-Date=20260528T015017Z&HW-CC-Expire=86400&HW-CC-Sign=849B911CE1EA8B3D431A5574D6F0547B1F4B78C1CE1EFE73BAA3701440AECC12)
 

  
如果部署的是多实例，升级前需要停下所有机器中的ohpm-repo服务，再进行升级操作。
2. 若想在其他目录使用ohpm-repo，请将对应版本ohpm-repo工具包解压目录中bin目录的路径配置到[系统环境变量](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/ide-ohpm-repo-faq#section24117279211)path中。
3. 下载并解压工具包：下载新版本的ohpm-repo私仓工具包，并解压（请解压到一个空文件夹中）。

  
![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/02/v3/7DHw4IO3TkCqIWs4LTW6ug/zh-cn_image_0000002571545588.png?HW-CC-KV=V1&HW-CC-Date=20260528T015017Z&HW-CC-Expire=86400&HW-CC-Sign=6E905058F8D94E3610C37309D5428D9A672B1F9CA0D62306E7BD295DFEDD30CA)

4. 安装完成之后，进入ohpm-repo私仓工具包解压目录下的bin目录，执行如下命令：

  
```text
ohpm-repo -v
```
 终端输出为新版本的版本号，则表示解压成功。
5. 移植配置文件信息：ohpm-repo的版本更新前后都保持在2.X.X/5.X.X范围内，新版本配置文件可以直接复用旧版本配置文件。

  
找到ohpm-repo旧版本的配置文件：如果旧版本未指定配置文件，使用默认配置文件：旧版本ohpm-repo解压根目录/conf/config.yaml。
6. 拷贝旧版本配置文件至新版本：复制旧版本的配置文件到新版ohpm-repo解压目录下的conf文件夹，覆盖新版本中的默认配置文件。
7. （可选项）新版本如果需要使用新的部署目录&lt;new_deploy_root&gt;，需要手动迁移数据。

  
升级ohpm-repo：按照步骤1-4，解压ohpm-repo并拷贝替换配置文件信息。
8. 建立新部署目录：判断指定的新部署目录&lt;new_deploy_root&gt;是否存在，不存在则新建，新部署目录需存在且为空。
9. 拷贝数据文件：拷贝旧版本部署目录&lt;deploy_root&gt;下的全部文件至新部署目录中。
10. 修改新版本ohpm-repo配置文件：打开新版本ohpm-repo的解压根目录，进入conf目录下，修改配置文件config.yaml中配置项deploy_root为新的部署目录&lt;new_deploy_root&gt;。
1. 新版本服务启动：正确拷贝替换配置文件信息后，进入ohpm-repo私仓工具包解压目录下的bin目录，执行以下命令启动新版本ohpm-repo服务。

  
执行安装命令：
```text
<em>ohpm-repo install</em>
```
 结果示例：

  
![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/5d/v3/Qci1M_XjTzK_2Cw5GOlTag/zh-cn_image_0000002571385952.png?HW-CC-KV=V1&HW-CC-Date=20260528T015017Z&HW-CC-Expire=86400&HW-CC-Sign=08F54AD3314A428A757C07DF110726FF9CC608275C85AA3CC6F84E72C7140109)

2. 刷新环境变量：安装成功后，必须根据给出的提示信息刷新环境变量，Windows系统和Linux/Mac 系统，有不同处理方式：
Windows系统： 关闭当前窗口，重新开启一个窗口。
3. Linux/Mac系统：

  
5.0.1之前版本：在命令行中执行刷新命令：source ~/.bashrc或者.~/.bashrc ；
4. 5.0.1及以后版本：

  在命令行中执行刷新命令：当shell为bash时，执行source ~/.bashrc或者. ~/.bashrc ；当shell为zsh时，执行source ~/.zshrc或者.~/.zshrc 。
5. 执行start命令：
```text
<em>ohpm-repo start</em>
```
 结果示例：

  
![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/2a/v3/BF4lMuu4ToKVdTT1GQ70PA/zh-cn_image_0000002602065071.png?HW-CC-KV=V1&HW-CC-Date=20260528T015017Z&HW-CC-Expire=86400&HW-CC-Sign=25DE055F6285B1E757B61CEAB96EE3BF3B232C4B0A25E2CAF01C756A9F977818)


  
> [!NOTE]
> 版本升级之前，如果浏览器中已访问ohpm-repo页面，版本升级之后请刷新ohpm-repo页面。

6. 多实例部署机器快速升级：在多实例部署中，其他机器同步骤1-6进行快速升级操作。
