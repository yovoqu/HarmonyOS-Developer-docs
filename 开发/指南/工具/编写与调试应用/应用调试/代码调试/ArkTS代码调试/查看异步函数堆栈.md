# 查看异步函数堆栈

更新时间：2026-01-15 06:51:04

来源：https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/ide-debug-async-stack-traces

从DevEco Studio 5.1.1 Beta1版本开始，开发者可通过打开异步堆栈跟踪开关、设置异步调用链深度来跟踪异步函数调用的顺序。
 
> [!NOTE]
> 异步堆栈跟踪开关为全局设置，开启后所有工程都生效。 修改异步堆栈跟踪开关或异步调用链深度后，需要重新启动调试或启动新的调试会话才会生效。 setTimeout函数异步堆栈不生效。 异步堆栈不展示变量列表。

1. 点击**File > Settings**（macOS为**DevEco Studio > Preferences/Settings**）** > Build, Execution, Deployment > Debugger > Async Stack Traces**。
勾选**Enable async stack traces**打开异步堆栈跟踪开关。
2. 设置异步调用链深度**Async call chain depth**大于0，才能在调试堆栈时展示调用链对应层数。
3. 在异步调用链中设置断点，启动调试，命中断点后，堆栈列表将展示对应调用链层数。如果实际的调用链层数比设置的异步调用链深度小，则只展示实际调用链层数。每个异步调用链以**Async call from**分隔，后面是调用函数。
