# 应用架构

更新时间：2026-03-11 06:16:00

来源：https://developer.huawei.com/consumer/cn/doc/design-guides/application-architecture-0000002202799801

应用需遵循一定的系统规范进行应用架构、应用导航的设计。
 

##### 应用架构

 
智能穿戴产品的信息布局需要根据当前应用的实际内容进行设计，根据手表界面信息的有无可规范为两大应用架构：非空界面通用架构和空界面架构。其中空界面架构，无需展示其他标题等导航类信息，使用简单图文告知当前界面状态即可。
  
|  |  |  |  |
| 标题栏+内容区 一般情况下，每一级非空界面需要通过标题明确告知用户当前所处的应用或层级。 | 顶部按钮+内容区 有需要新建、添加等功能操作时，可使用圆形按钮放在顶部，按钮可支持悬浮。 | 按钮+文字描述 有操作的空界面，如新建、添加等，可直接在界面中提供操作和描述。 | 图标+文字描述 不带操作的空界面，界面中心展示图标和描述。 |
 
 

##### 应用导航

 
导航主要用于引导用户路径以及告知当前的位置，目前智能穿戴产品的应用导航类型主要分为两类：层级导航、平级导航。
 

##### 层级导航

 
层级导航结构由父页面和子页面组成。父页面可以有一个或多个子页面。层级导航有利于我们呈现具有包含关系的复杂菜单内容，另外层级导航也承担入口引导的作用。根据应用功能的主次关系，手表的层级导航有两种
  
|  |  |
| 多功能菜单为首页 应用可以向用户展示应用的功能列表，从父页面进入后，各个子页面互相不冲突。 | 主功能为首页 应用可以清晰的向用户展示首页的主线功能，通过屏幕右滑至菜单页面，可快捷导航至次要功能页面。 |
 
 

##### 平级导航

 
平级内容展示超过一屏时，可考虑把更多内容切换至下一屏进行独立布置，以防止内容平铺导致的圆形屏幕边缘的信息丢失。根据智能穿戴设备的屏幕交互和空间延展特性，我们将平级导航分为三类：横向切屏导航、垂直切屏导航、垂直平铺导航。
  
| 横向切屏导航 平级内容展示超过一屏时，可考虑横向把更多内容切换至下一屏进行独立布置，以防止内容平铺导致的圆形屏幕边缘的信息丢失。 |  |
| 垂直切屏导航 平级内容展示超过一屏时，可考虑把更多内容垂直切换至下一屏进行独立布置，即使用垂直切屏导航。该导航类型极大拓展了手表上下信息承载的空间。 |  |
| 垂直平铺导航 平级内容展示超过一屏时，可考虑把更多的内容平铺在延长的纵向页面，即使用垂直平铺导航。该导航类型区别于垂直切屏导航可以更连贯的展示信息内容。 |  |
 
 

##### 如何使用

 
在实际应用中，除了单一应用某种导航类型，组合使用也较为常见。部分复杂功能往往会使用一到两种导航类型，且较多使用层级导航与平级导航的组合。
  
|  |  |
| 以手表“锻炼记录”应用示例，“锻炼记录”应用使用了层级导航和垂直切屏导航两种类型。 | 以手表“华为音乐”应用为例，“华为音乐”应用的“推荐”页面为首页，右滑进入次要功能导航页面。 |
