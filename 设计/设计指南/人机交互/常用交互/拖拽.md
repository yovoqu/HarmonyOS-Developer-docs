#### 拖拽的定义

拖拽是在应用程序内部、在应用程序之间传输数据的可视化操作方式。 拖拽操作可以让用户使用标准手势 (用手指按住并移动、用鼠标按住并移动、触控笔按住并移动) 在应用程序之间、在应用程序内部传输数据。拖拽操作基于移动和复制的目的，如文件管理器中将内容拖拽到文件夹内为移动，网页中的照片拖拽到记事本中为复制。开发相关描述请参阅[拖拽事件](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/arkts-common-events-drag-event-V5)文档。


| 场景 | 界面 | 举例 |
| --- | --- | --- |
| 同一应用页面 | 输入编辑界面，在不同输入区域或同一输入区域的不同位置。 | 如备忘录、记事本、联系人、邮件中移动文本或图片等。 |
|    | 排序或移动位置。 | 桌面编辑界面或天气城市编辑界面。 |
| 同一应用不同页面 | 分栏情况下用拖拽的方式来切换页面来放到不同的页面。 | 备忘录中把文本从一条备忘录中放到另外一条备忘录中。 |
| 跨模块 | 分屏、悬浮窗场景。 | 从一个应用的界面拖拽到另外一个应用的界面。如从一个分屏界面拖拽到另外一个分屏界面。从悬浮窗窗口拖拽到全屏界面。 |
| 跨系统 | 电脑协同 | 从电脑到手机或手机到 电脑 |



#### 拖拽的对象

拖拽对象：文本、图片、视频、文件、链接、列表项、宫格项、卡片、浏览器文稿页签等等。

用户可以拖拽一个或多个对象。也可以同时拖拽不同数据对象，例如文本 + 图片、图片 + 视频等，具体支持拖拽的对象根据业务场景定义。

以下控件支持拖拽，主要有两类控件文本类和内容类，调用系统控件则支持发起拖拽、拖拽中、释放拖拽的全流程交互，应用只需要适配数据处理能力，控件拖入拖出能力如下表。

#### 拖拽的流程

| 拖拽的流程为发起拖拽、拖拽中、释放拖拽三个阶段。用户可通过多种输入源发起拖拽，不同输入源的拖拽操作会有一定差异。 |  |




#### 发起拖拽




拖拽常用的输入方式为触屏和鼠标指针，选中被拖拽的对象后就能发起拖拽。


#### 触屏拖拽


触屏发起拖拽首先要选中对象，其次长按移动后触发拖拽对象，移动大于 10 vp 则可发起拖拽，移动后对象会进行适当缩放不遮挡页面，如果拖拽失败，原有的选中态仍然保留。

| 选中对象 文本类对象需要先进入选中态后才能发起拖拽。内容类对象长按即可直接选中发起拖拽。 |  |
| 长按浮起 长按拖拽对象浮起可拖拽对象的预览图，并伴随着震动反馈。 |  |
| 长按悬浮菜单和拖拽的关系 1.长按内容浮起 长按后移动可直接发起拖拽 2.预览内容可中断 一直长按会出现预览内容，在预览内容出现前可被打断。 3.一直长按可出现预览菜单，不抬手对预览内容可直接发起拖拽，抬手后也可以直接发起拖拽。 |  |
| 移动发起拖拽 移动后完成发起拖拽的行为，原对象灰显 |  |


**拖拽对象的显示缩放规则**

为了保证拖出的对象对内容不造成遮挡，提供缩放的统一规则。 框架也提供对象是否缩放的设置项，通常在排序的场景下不需要缩放，如桌面卡片的拖拽排序。关于拖拽对象的复制剪切行为建议都使用默认的缩放规则。

| 对象不缩放跟手移动 | 对象缩放 | 对象插入文本光标下位移 |
|  |  |  |
| 对象不需要缩放的场景下，对象保持原大小，拖拽交互时跟手移动。 | 对象较大时，需等比例缩放 | 光标在拖拽对象的上方显示 手指在拖拽对象的顶部居中的位置 |


**多个对象拖出**

选择多个对象可以同时拖出。

 - 长按某个勾选项，手指长按的图堆叠在最上方。
 - 长按某个未勾选项，该未勾选项变成勾选，且和已勾选项聚拢成一堆。


|  |  |
| 多选聚拢效果 | 多选聚合带有数量角标 |


**云端文件处理**

拖拽对象中如果是云端文件，如云端的图片或视频，仍然可以拖拽。等到目的地再进行下载。




#### 鼠标拖拽


鼠标发起拖拽首先要选中对象，其次按下鼠标移动后触发拖拽对象，移动后对象会进行适当缩放不遮挡页面，发起拖拽时误丢手了原来的选中态仍然保留。

| 选中对象 文本类对象需要先进入选中态后才能发起拖拽。内容类对象按住即可直接选中发起拖拽。 |  |
| 移动拖拽 选中对象后，移动即可拖拽，原对象显示灰色。 |  |
| 丢手选中态保留 浮起选中内容后直接丢手，选中状态保留。 复制拖拽后，原元素选中状态保留。 |  |


**多个元素拖拽**

多选多个元素拖拽，开启自动布局则勾选项聚拢。

多选多个元素拖拽，不开启自动布局则勾选项不聚拢，平铺预览拖拽。

|  |  |
| 多选对象默认聚拢右上角显示拖拽数量角标 | 应用可配置桌不聚拢直接拖拽，拖拽状态显示在角标下方 |

**复制和移动**

对象可以被复制也可以被剪切移动，复制对象则原对象显示灰色。

| 复制 | 移动 |
|  |  |

**多指拖拽**

拖拽过程中支持多指触屏交互，如拖拽过程中一只手按住拖拽对象，使用另一只手指滑动屏幕。拖拽过程中可使用另一只手切换应用等。

|  |    |
| 多指滚屏 |    |


**拖拽抵达边缘可触发滚屏**

| 拖拽抵达页面边缘后可触发滚屏 |  |
| 当拖拽对象进入文本编辑界面时，当光标处于屏幕可见区域的第一行或者最后一行时，用户继续往上或往下挪动，页面往下或往上滚动以便用户插入光标的位置。 |  |
| 移动拖拽 将对象拖拽至相应的区域后，右上角角标状态基于实际状态进行切换，提示用户状态的变化。有移动、复制的单个或多个的状态。若该区域不接收拖拽或因格式问题无法接收当前拖拽的对象，则呈现为移动状态。 |  |



#### 鼠标拖拽中


| 键鼠可配合同时拖拽 拖拽时按住 Ctrl 键可以将移动内容更改为复制。 不同窗口间的拖拽通常为复制行为。应用场景可根据实际体验设计是移动还是复制，如文管、桌面中跨不同层级界面拖拽时，是同一内容的移动，便于文件管理。 |
| 拖拽中格式变化 鼠标拖拽时，根据指针变化的位置，对象样式变成接纳对象的目标格式。 |
|  |



#### 释放拖拽


当拖拽对象处于可接受的状态时，丢手即执行释放动作，添加至对应的位置上。拖拽对象不被接受则插入失败。

| 拖拽成功 释放拖拽对象，对象变成目标窗口格式，目标对象可接纳拖拽对象的格式由接受的场景定义。 在编辑输入类区域，表现为插入缩略图。 非编辑区域，丢手后的处理机制根据业务情况定义。如拖拽图片到聊天界面是直接发送图片。 |  |


**拖拽失败**

目标对象不支持拖拽对象的插入，预览图右上角会出现禁止角标。

| 拖拽目标对象处于拖拽操作的可见范围时，抬手后拖拽对象归位到原位置。 | 拖拽目标对象处于拖拽操作的不可见范围时，如跨窗口拖拽时，抬手后拖拽对象原地放大淡出。 |
|  |  |


**进度处理**

当拖拽内容过多时，松手后应用接收，如有响应导致的停滞现象，在等待的这段时间，对象通过呼吸缩放动效缓解等待感。目标应用处理需要耗费的时间过长，此时则需要给出等待时间提示。
