# 智能穿戴应用 UX 体验标准

更新时间：

来源：https://developer.huawei.com/consumer/cn/doc/design-guides/ux-guidelines-wearable-0000002503475023

此标准为智能穿戴（含圆形屏、方形屏）特有体验标准，除了此标准，应用开发者也需要遵循通用应用体验标准中智能穿戴设备适用的部分
 

 
8.1 界面布局
  
| 标准编号 | 8.1.1 | 圆形屏边距 |
| 标准描述 | 为避免文本与按钮在屏幕内截断与难以点击，设置安全边距，建议圆形屏应用使用文本居中对齐： 元素距离屏幕左侧间距 26vp 元素距离屏幕右侧间距 26vp 元素距离屏幕顶部间距 20vp 元素距离屏幕底部间距 6vp 长页面滑动到底部的边距 40vp |
| 测试方法 | 应用在支持的所有设备类型上打开，获取应用上下左右的间距信息，观察间距是否满足要求。 |
| 判定标准 | 应用全屏情况下，观察关键交互功能是否避让安全间距 |
| 标准等级 | 必须 |
| 适用设备类型 | 智能穿戴 |
| 需考虑的特殊事项 | 无 |
| 系统能力 | 无系统能力，请遵循设计规范 |
 
  
| 标准编号 | 8.1.2 | 方形屏边距 |
| 标准描述 | 为避免文本与按钮在屏幕内截断与难以点击，设置安全边距，建议方形屏应用使用文本左对齐： 元素距离屏幕左侧间距 12vp 元素距离屏幕右侧间距 12vp 元素距离屏幕顶部间距 12vp 元素距离屏幕底部间距 6vp 长页面滑动到底部的边距 36vp。 |
| 测试方法 | 应用在支持的所有设备类型上打开，获取应用上下左右的间距信息，观察间距是否满足要求。 |
| 判定标准 | 应用全屏情况下，观察关键交互功能是否避让安全间距 |
| 标准等级 | 必须 |
| 适用设备类型 | 智能穿戴 |
| 需考虑的特殊事项 | 无 |
| 系统能力 | 无系统能力，请遵循设计规范 |
 
 
8.2 导航点
  
| 标准编号 | 8.2.1 | 圆形屏导航点 |
| 标准描述 | 页面有不同的子类别，内容一屏显示不截断，为了保证页面导航清晰，使用Swiper导航点。如图示，横向布局页面使用横向导航点，纵向布局页面使用竖向导航点。 |
| 测试方法 | 应用在支持的所有设备类型上打开，获取应用上下切屏场景，观察横/纵向Swiper导航点是否满足要求。 |
| 判定标准 | 应用初始进入和切屏情况下，观察页面是否有Swiper导航点，点的数量与页面数量一一对应，且能同步切换。 |
| 标准等级 | 必须 |
| 适用设备类型 | 智能穿戴 |
| 需考虑的特殊事项 | 无 |
| 系统能力 | 请参阅ArcSwiper |
 
  
| 标准编号 | 8.2.2 | 方形屏导航点 |
| 标准描述 | 页面有不同的子类别，内容一屏显示不截断，为了保证页面导航清晰，使用Swiper导航点。如图示，横向布局页面使用横向导航点，纵向布局页面使用竖向导航点。 纵向导航点：垂直居中，宽度5vp，间隔4vp，主焦点长度14vp，边距为2vp。 横向导航点：水平居中，高度5vp，间隔4vp，主焦点长度10vp，边距为6vp。 |
| 测试方法 | 应用在支持的所有设备类型上打开，获取应用上下切屏场景，观察横/纵向Swiper导航点是否满足要求。 |
| 判定标准 | 应用初始进入和切屏情况下，观察页面是否有Swiper导航点，点的数量与页面数量一一对应，且能同步切换。 |
| 标准等级 | 必须 |
| 适用设备类型 | 智能穿戴 |
| 需考虑的特殊事项 | 无 |
| 系统能力 | 无系统能力，请遵循设计规范 |
 
 
8.3 圆形屏列表缩放
  
| 标准编号 | 8.3 | 圆形屏列表缩放 |
| 标准描述 | 页面有2个以上长列表时，使用缩放效果。 |
| 测试方法 | 应用在支持的所有设备类型上打开，获取应用长列表场景，观察列表缩放能力是否满足要求。 |
| 判定标准 | 中心列表缩放固定位108% 其余列表距离中心点越远，缩放值越小。 |
| 标准等级 | 必须 |
| 适用设备类型 | 智能穿戴 |
| 需考虑的特殊事项 | 无 |
| 系统能力 | 请参阅ArcList |
 
 
8.4 应用图标
  
| 标准编号 | 8.4 | 应用图标 |
| 标准描述 | 智能穿戴应用图标需适配为圆形，不分层，周围各添加1px内间距，整体尺寸为152px*152px |
| 测试方法 | 获取应用列表、消息通知内图标形状和尺寸信息，观察应用图标在设备上的显示效果。 |
| 判定标准 | 应用图标显示正常，图标显示完整。无明显的模糊、拉伸、压缩、锯齿等情况。 |
| 标准等级 | 必须 |
| 适用设备类型 | 智能穿戴 |
| 需考虑的特殊事项 | 无 |
| 系统能力 | 设计规范 |
