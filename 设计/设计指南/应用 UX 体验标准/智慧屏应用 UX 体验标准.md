# 智慧屏应用 UX 体验标准

更新时间：

来源：https://developer.huawei.com/consumer/cn/doc/design-guides/ux-guidelines-vision-0000002360813381

此标准为智慧屏特有体验标准，除了此标准，应用开发者也需要遵循通用应用体验标准和大屏应用体验标准中智慧屏适用的部分
 

 
6.1 边距
  
| 标准编号 | 6.1 | 边距 |
| 标准描述 | 为避免指向遥控器在屏幕边缘难以点击，设置安全边距 元素距离屏幕左侧间距 56vp 元素距离屏幕右侧间距 56vp 元素距离屏幕顶部间距 22vp 元素距离屏幕底部间距 22vp |
| 测试方法 | 应用在支持的所有设备类型上打开，获取应用上下左右的间距信息，观察间距是否满足要求。 |
| 判定标准 | 应用全屏情况下，观察关键交互功能是否避让安全间距 |
| 标准等级 | 必须 |
| 适用设备类型 | 智慧屏 |
| 需考虑的特殊事项 | 无 |
| 系统能力 | 设计规范 |
 
 
6.2 沉浸型布局
  
| 标准编号 | 6.2 | 沉浸型布局 |
| 标准描述 | 沉浸型、或具备富媒体内容的应用或页面，例如视频、音乐应用的首页，建议使用沉浸式大图 + 顶 Tab 的布局效果，提升页面的视觉冲击力与沉浸感 |
| 测试方法 | 应用在支持的所有设备类型上打开，检查富媒体内容相关页面： 页面通过沉浸式大图/视频+顶 Tab 布局设计提升视觉创新。图片/视频满足4K清晰度。 |
| 判定标准 | 启动满足条件的应用，观察其页面是否使用沉浸式布局。 |
| 标准等级 | 推荐 |
| 适用设备类型 | 智慧屏 |
| 需考虑的特殊事项 | 无 |
| 系统能力 | 设计规范 |
 
 
6.3 动效创新
  
| 标准编号 | 6.3 | 动效创新 |
| 标准描述 | 结合指向遥控器的悬停交互方式，有动效提升的创新，带来更生动、丝滑的操作体验。 |
| 测试方法 | 遥控器指向应用内控件并移动，检查控件交互反馈： 控件支持 3D 扭转等创新动效；动效流畅。 |
| 判定标准 | 观察被指向内容是否根据 悬浮状态、光标移动等呈现联动的动态效果 |
| 标准等级 | 推荐 |
| 适用设备类型 | 智慧屏 |
| 需考虑的特殊事项 | 无 |
| 系统能力 | 请参阅悬浮事件 |
 
 
6.4 光标悬停反馈
  
| 标准编号 | 6.4 | 光标悬停反馈 |
| 标准描述 | 所有可交互控件需对指向遥控器光标的悬停状态作出视觉反馈。 |
| 测试方法 | 应用在支持的所有设备类型上打开，将光标悬浮在可交互控件上，检查控件的视觉反馈符合要求： 视觉反馈存在：光标悬浮时，检查可交互控件外观出现视觉反馈；视觉反馈适当：观察控件的视觉反馈，应符合当前交互行为语义，如高亮、尺寸变大。 |
| 判定标准 | 控件出现视觉反馈，提示控件的可交互性以及当前交互行为语义。 |
| 标准等级 | 必须 |
| 适用设备类型 | 智慧屏 |
| 需考虑的特殊事项 | 无 |
| 系统能力 | 请参阅悬浮事件 |
