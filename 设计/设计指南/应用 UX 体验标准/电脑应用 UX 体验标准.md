# 电脑应用 UX 体验标准

更新时间：2026-04-15 07:47:30

来源：https://developer.huawei.com/consumer/cn/doc/design-guides/ux-guidelines-2in1-0000001777895636

此标准为电脑特有体验标准，除了此标准，应用开发者也需要遵循通用应用体验标准和大屏应用体验标准中电脑适用的部分。
 

#### 窗口响应式

5.1.1 支持窗口化运行
  
| 标准编号 | 5.1.1 | 支持窗口化运行 |
| 标准描述 | 除游戏应用外，应用支持窗口化运行。应用打开时建议以系统默认窗口尺寸进行显示。若应用结合自身业务，有不同于系统默认窗口尺寸的诉求，可按需修改默认尺寸。 |
| 测试方法 | 打开应用，观察窗口大小。 |
| 判定标准 | 以系统默认窗口尺寸显示。 |
| 标准等级 | 推荐 |
| 适用设备类型 | 电脑 |
| 需考虑的特殊事项 | 无 |
| 系统能力 | 系统能力具备 |
 
 
5.1.2 支持窗口形态转换
  
| 标准编号 | 5.1.2 | 支持窗口形态转换 |
| 标准描述 | 在电脑打开应用，通过窗口控制器调整窗口形态： 1.除游戏外，应用支持通过窗口控制器在全屏显示/最大化窗口、最小化窗口、关闭窗口之间进行转换。最小化窗口和关闭窗口功能为必选，最大化窗口功能推荐。 2.窗口形态转换按钮的顺序从左至右为最大化、最小化、关闭。 |
| 测试方法 | 通过窗口控制器调节应用窗口大小。 |
| 判定标准 | 1. 应用支持通过窗口控制器，在全屏显示/最大化窗口、最小化窗口、和其他窗口尺寸之间进行转换（除游戏外）。最小化窗口和关闭窗口功能为必选，最大化窗口功能推荐。 2. ﻿﻿窗口形态转换按钮的顺序从左至右为最大化，最小化，关闭。 |
| 标准等级 | 必须 |
| 适用设备类型 | 电脑 |
| 需考虑的特殊事项 | 无 |
| 系统能力 | 请参阅多设备窗口形态-窗口沉浸式 |
 
 
5.1.3 支持窗口尺寸调节
  
| 标准编号 | 5.1.3 | 支持窗口尺寸调节 |
| 标准描述 | 应用支持在最小窗口尺寸到鼠标可拖拽到达的最大窗口尺寸之间任意调节窗口大小，并保证应用界面内容实时响应窗口尺寸变化，应用界面布局不出现显示异常错乱问题。 |
| 测试方法 | 自由调节窗口大小，观察应用窗口与界面内容。 |
| 判定标准 | 应用支持在最小窗口尺寸到最大窗口尺寸之间任意调节。应用界面内容实时响应窗口尺寸变化，无显示异常。 |
| 标准等级 | 推荐 |
| 适用设备类型 | 电脑 |
| 需考虑的特殊事项 | 无 |
| 系统能力 | 请参阅自适应布局 |
 
 
5.1.4 支持分屏和比例调节
  
| 标准编号 | 5.1.4 | 支持分屏和比例调节 |
| 标准描述 | 应用支持分屏显示，并支持通过拖动条调整窗口显示比例，应用界面内容响应窗口比例变化，确保能正常显示。 |
| 测试方法 | 让应用进入分屏状态，并调整显示比例，观察应用窗口与界面内容。 |
| 判定标准 | 应用支持分屏显示，窗口比例调节过程中无显示异常。 |
| 标准等级 | 推荐 |
| 适用设备类型 | 电脑 |
| 需考虑的特殊事项 | 无 |
| 系统能力 | 请参阅一多开发实例（购物比价）-商品详情页 |
 
 
5.1.5 窗口内容状态保持
  
| 标准编号 | 5.1.5 | 窗口内容状态保持 |
| 标准描述 | 应用窗口在进行尺寸调节、形态转换时应保持窗口内容的状态，如已键入的文本、已选中的文本或图片、已选中的项目（列表、宫格等）。 |
| 测试方法 | 调节应用窗口尺寸和窗口状态，观察窗口内容。 |
| 判定标准 | 窗口内容状态未发生变化。 |
| 标准等级 | 必须 |
| 适用设备类型 | 电脑 |
| 需考虑的特殊事项 | 无 |
| 系统能力 | 系统能力具备，无需开发者关注 |
 
 
5.1.6 折叠电脑应用子窗跟随主窗
  
| 标准编号 | 5.1.6 | 折叠电脑应用子窗跟随主窗 |
| 标准描述 | 在折叠态时，应用子窗需遵循双屏独立反馈原则，即从上/下屏触发则从上/下屏反馈，同时避让中轴区域。 |
| 测试方法 | 打开应用，观察子窗是否与主窗在同一所在屏。 |
| 判定标准 | 1. 折叠态时，当主窗位于上屏时，打开的子窗需同样位于上屏。 2. 折叠态时，当主窗位于下屏时，打开的子窗需同样位于下屏。 |
| 标准等级 | 必须 |
| 适用设备类型 | 折叠电脑 |
| 需考虑的特殊事项 | 无 |
| 系统能力 | 系统能力具备，无需开发者关注 |
