更新时间：2026-03-11 06:16:00

##### 功能完整

3.1.1 横竖屏适配
  
| 标准编号 | 3.1.1 | 横竖屏适配 |
| 标准描述 | 应用支持在折叠屏展开态、平板设备上的横竖屏切换显示。 屏幕比例接近 1:1 的设备，横竖屏布局需保持一致。 折叠屏的横屏响应式布局适配示例 平板的横屏响应式布局适配示例 比例接近 1:1 的屏幕上横竖屏布局一致的示例。 |
| 测试方法 | 设备旋转，观察横竖屏切换旋转过程和横竖屏的界面布局。 |
| 判定标准 | 适配横屏显示，横竖屏切换过程流畅，不卡顿；屏幕比例接近 1:1 时横竖屏布局正常且一致。 |
| 标准等级 | 必须 |
| 适用设备类型 | 折叠屏、平板、电脑 |
| 需考虑的特殊事项 | 无 |
| 系统能力 | 请参阅自适应布局 |
 
 
3.1.2 多窗适配
  
| 标准编号 | 3.1.2 | 多窗适配 |
| 标准描述 | 应用适配竖向悬浮窗、左右分屏、上下分屏，且确保该形态下有较好的适配效果，能完成全流程交互。例如上下分屏时，不出现弹出框过大无法点击关闭按钮，导致交互中止等问题；不出现内容显示过大导致交互效率低浏览体验差等问题。 视频、游戏类应用需要适配横向悬浮窗。应用在平板设备上需适配支持窗口大小变化调整，确保应用内容显示无异常。 竖向悬浮窗、横向悬浮窗、左右分屏、上下分屏的示例。 |
| 测试方法 | 在设备上启动悬浮窗、分屏，观察是否能正常触发多窗，多窗的显示布局是否正常，应用是否正常运行。 |
| 判定标准 | 设备能正常启动悬浮窗、分屏，且悬浮窗和分屏内布局正常，不出现卡顿、闪退、闪屏、变形、错位等情况。 |
| 标准等级 | 必须 |
| 适用设备类型 | 折叠屏、平板、电脑、智慧屏 |
| 需考虑的特殊事项 | 无 |
| 系统能力 | 1.悬浮框，请参阅管理应用窗口（Stage 模型） 2.分屏，请参阅一多开发实例（购物比价）-商品详情页 |

3.2.1 布局合理
 
3.2.1.1 布局基础要求
  
| 标准编号 | 3.2.1.1 | 布局基础要求 |
| 标准描述 | 设备在折叠/展开或横竖屏切换时，应用中的图片、视频等界面元素应避免出现错位/截断/变形/模糊等问题。 |
| 测试方法 | 在大尺寸屏幕 (折叠屏/平板) 设备上，检查应用的界面显示。 |
| 判定标准 | 应用的界面布局正常，图片、视频等界面元素不会发生错位/截断/变形/模糊等问题，关键信息内容展示正常。 |
| 标准等级 | 必须 |
| 适用设备类型 | 折叠屏、平板、电脑 |
| 需考虑的特殊事项 | 无 |
| 系统能力 | 1.折叠状态监听，请参阅@ohos.display-display.on('foldStatusChange')10+ 2.响应式/自适应布局，请参阅自适应布局 |
 
 
3.2.1.2 图标文字大小适中
  
| 标准编号 | 3.2.1.2 | 图标文字大小适中 |
| 标准描述 | 文字/图标物理大小建议： 双折叠/三折叠设备展开态文字、图标大小为折叠态的1~1.2倍 (推荐)。 平板上文字/图标大小为直板机的1~1.5倍（推荐）。 文字/图标放大倍数不超过1.2/1.5倍 (必须)。 不建议一排图标数量过多导致信息过密: 1. 双折叠设备展开态横竖屏/三折叠设备双屏态横竖屏/三折叠设备三屏态竖屏/平板竖屏上建议一排不超过8个图标（推荐）。 2. 三折叠设备三屏态横屏/平板横屏上建议一排不超过13个图标（推荐）。 3. 智慧屏建议一排不超过12个图标（推荐）。 Do (图标间距均匀变大+挪移布局) Don't (信息过密) |
| 测试方法 | 在设备的折叠展开、横竖屏旋转状态下，观测并获取文字和图标的大小和界面显示。 |
| 判定标准 | 展开态文字/图标的大小和数量不超过阈值。 |
| 标准等级 | 必须 |
| 适用设备类型 | 折叠屏、平板、电脑、智慧屏 |
| 需考虑的特殊事项 | 无 |
| 系统能力 | 1.图标尺寸，请参阅尺寸设置 2.文字尺寸，请参阅文本通用-fontSize |
 
 
3.2.1.3 弹出框大小适中
  
| 标准编号 | 3.2.1.3 | 弹出框大小适中 |
| 标准描述 | 双折叠设备/三折叠设备展开态弹出框高度为折叠态的1~1.2倍 (推荐)。 平板上弹出框高度为直板机的1~1.2倍（推荐）。 弹出框高度放大倍数不超过1.2倍 (必须)。 电脑设备弹出框最小不低于尺寸（360*240vp），最大不超过当前应用窗口尺寸。 智慧屏设备弹出框宽度<36%屏幕总宽度，高度<80%屏幕总高度。 弹出框大小适中的示例 |
| 测试方法 | 在设备的折叠展开、横竖屏旋转状态下，观测并获取弹出框的大小。 |
| 判定标准 | 1. 展开态或平板上弹出框的高度为折叠态/直板机高度的 1~1.2 倍。 2. 电脑设备弹出框高度为最小尺寸（360*240vp）到当前应用窗口尺寸之间。 3. 智慧屏弹窗框宽度与高度不超过屏幕比例阈值。 |
| 标准等级 | 必须 |
| 适用设备类型 | 折叠屏、平板、电脑、智慧屏 |
| 需考虑的特殊事项 | 无 |
| 系统能力 | 1.判断当前折叠状态，请参阅@ohos.display.on('foldStatusChange')10+ 2.设置自定义弹窗尺寸，请参阅自定义弹窗-CustomDialogControllerOptions对象说明 |

3.2.1.6 上下图文信息量适中
  
| 标准编号 | 3.2.1.6 | 上下图文信息量适中 |
| 标准描述 | 图片放大倍数不超过规范要求且控件高度占比满足规范要求（推荐）； 图片放大倍数不超过规范要求或控件高度占比满足规范要求（必须）。 1. 上下图文结构中，建议信息流场景的图片左对齐，阅读场景的图片居中对齐。 2. 图片的物理尺寸：双折叠/三折叠设备展开态上图片等比放大时，放大倍数建议不超过1.2倍，平板上比直板机放大不超过1.5倍。 3. 在平板、双折叠/三折叠设备展开态、电脑设备最大化窗口等宽屏设备上最好通过延伸布局、挪移布局等方式让图文进行合理布局，避免图片过大。 上下图文场景中图片高度占比要求如下： 1. 信息流场景中，非宫格聚合图片高度不要超过屏幕高度的40%，长图或宫格聚合图片整体高度不要超过屏幕高度的60%。 2. 内容详情页中，图片高度不要超过屏幕高度的60%。 首页的入口型上下图文示例 详情页上下图文示例 |

| 标准描述 | 页面布局满足折叠屏/平板等大屏设备持握时拇指的易操作性，高频操作的控件，例如导航、悬浮按钮、工具栏的核心操作等，避开难交互区域。 多端的易交互热区分布 1 易交互 2 可交互 3 难交互 |
| 测试方法 | 观察应用内的导航、悬浮按钮、工具栏的核心操作等的位置是否满足在拇指易交互区域。 |
| 判定标准 | 导航、悬浮按钮、工具栏的核心操作位于拇指易交互区域，则满足要求。 |
| 标准等级 | 推荐 |
| 适用设备类型 | 折叠屏、平板 |
