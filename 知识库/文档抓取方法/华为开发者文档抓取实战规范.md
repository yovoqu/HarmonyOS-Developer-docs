# 华为开发者文档抓取实战规范

## 目的

这篇文档专门总结华为开发者官网文档抓取时的实战规则，不讨论通用方法论，重点讲：

- 华为站点的页面特点
- 实际抓取策略
- 常见异常和处理规则

## 一、先看官网实际结构，不要只看任务描述

华为开发者文档抓取时，第一原则是：

- 以官网主目录页和子目录页的实际结构为准

原因是同一个模块的结构差异可能很大。

例如：

- `UI Design Kit` 可以拆成 `ArkTS API` 和 `ArkTS组件`
- `Ability Kit` 有明显的多级栏目结构
- `Accessibility Kit` 实际只有 `ArkTS API` 和 `错误码`

所以流程应当是：

1. 先打开主目录页
2. 确认一级栏目
3. 再进入每个栏目页确认二级结构
4. 最后再落本地目录

## 二、优先使用 Playwright

在华为开发者官网上，主抓取方案应当是：

- `Playwright`

不建议优先依赖：

- `curl`
- 直接复刻接口请求

因为常见问题包括：

- 连接重置
- connect timeout
- 页面动态渲染
- 接口上下文不完整

浏览器方式更接近真实访问链路，成功率更高。

## 三、页面等待策略

华为官网不适合只等某个固定加载事件。

建议策略：

1. `goto(..., waitUntil: "commit")`
2. 轮询检查：
   - `.doc-title`
   - `.markdown-body`
3. 直到标题或正文出现实际内容

判断标准要尽量务实：

- 标题非空
- 正文容器存在
- 正文 HTML 非空

## 四、目录设计规范

### 1. 目录先建最终结构

不要先放临时目录再搬运。

应该直接建立最终目录，例如：

- `应用框架/UI Design Kit/...`
- `应用框架/Ability Kit/...`
- `应用框架/Accessibility Kit/...`

### 2. 按栏目分目录

如果官网主目录存在子栏目，就按子栏目建目录。

如果官网栏目下还有明显的二级栏目，也继续保留层级。

### 3. 根目录建议保留 landing page

例如：

- `ability-api.landing.html`
- `accessibility-api.landing.html`

这样后续回看目录结构时更直观。

## 五、文件命名规范

默认命名规则：

- 使用官网 URL 尾部 slug
- 正文页：`slug.html`
- 栏目页或主目录页：`slug.landing.html`

例如：

- `ability-api.landing.html`
- `ability-arkts.landing.html`
- `js-apis-accessibility.html`

## 六、异常链接处理规则

华为官网文档中可能出现两类异常：

### 1. 异常 slug

表现为：

- URL 命名不符合同组规律
- 链接前缀缺字母
- 页面能打开，但 slug 明显不规范

处理方式：

- 如果页面有效，可以按同组命名规则修正本地文件名
- manifest 里同步更新 `slug` 和 `file`

### 2. 404 页面

表现为：

- 页面能打开，但标题是 `404`
- 或正文不是目标文档内容

处理方式：

- 不保留该 HTML
- 从 manifest 中剔除

## 七、manifest 规范

华为文档抓取建议统一输出 `manifest.json`。

推荐字段：

- `group`
- `slug`
- `kind`
- `file`
- `url`
- `title`

### 字段说明

- `group`：目录分组，例如 `Stage模型能力的接口`
- `kind`：`landing` 或 `doc`
- `file`：本地绝对路径
- `url`：官网来源地址

## 八、不同模块的推荐策略

### 1. UI Design Kit

推荐：

- 固定清单抓取

原因：

- 范围相对明确
- 页面名称和模块边界较清晰

### 2. Ability Kit

推荐：

- 按栏目页递归抓取

原因：

- 栏目分层明显
- 每个栏目页下挂很多正文页

### 3. Accessibility Kit

推荐：

- 少量固定页面直接抓取

原因：

- 实际页面规模小
- 结构非常简单

## 九、收尾检查规范

抓取完成后，至少要做这些检查：

1. 各目录文件数是否合理
2. manifest 总数是否与目录基本对应
3. 是否保留了 landing page
4. 是否存在 404 页面
5. 是否存在异常 slug
6. 是否有文件路径还指向旧目录

## 十、清理规范

任务完成后保留：

- 最终 HTML
- `manifest.json`

任务完成后删除：

- 临时脚本
- 调试文件
- 分片清单
- 缺失 URL 列表

## 十一、推荐执行顺序

1. 打开官网主目录页确认结构
2. 创建最终目录
3. 先保存主目录 landing page
4. 再保存栏目 landing page
5. 按栏目抓正文
6. 统一生成 HTML
7. 生成 manifest
8. 处理异常 slug
9. 剔除 404 页面
10. 清理中间文件
