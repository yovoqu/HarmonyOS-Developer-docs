# 动态页面处理（Playwright 备用方案）

> **注意**：当前主方案是直接调用 API。本文仅在 API 不可用时作为备用参考。

## 适用场景

- API 返回 `document not found` (code 92531031)
- 页面内容依赖滚动触发加载
- 图片、代码块需要滚动后才进入 DOM
- 目录树、折叠块影响正文可见性

## 核心原则

不是等"页面加载完"，而是等"正文可读"。

不依赖 `domcontentloaded`、`load`、`networkidle`。判断标准是：
- 正文标题已出现
- 正文容器存在
- 正文文本长度达到合理阈值
- 关键章节标题已出现

## 标准流程

```python
# 1. 打开页面
await page.goto(url, wait_until="commit", timeout=45000)

# 2. 等待正文可读
await page.wait_for_function("""
    () => {
        const title = document.querySelector('h1, .doc-title');
        const body = document.querySelector('.markdown-body, article, main, [role="main"]');
        return title && title.textContent.trim().length > 0 &&
               body && body.textContent.trim().length > 200;
    }
""", timeout=45000)

# 3. 分段滚动（很多页面滚动后才加载完整内容）
await page.evaluate("""
    () => new Promise(resolve => {
        let total = 0;
        const distance = 600;
        const timer = setInterval(() => {
            window.scrollBy(0, distance);
            total += distance;
            if (total >= document.body.scrollHeight - window.innerHeight) {
                clearInterval(timer);
                resolve();
            }
        }, 300);
    })
""")

# 4. 展开折叠内容
# 点击正文区域的"展开"、"Show more"等控件
# 不点击"下一篇"、"登录"等导航控件

# 5. 等待内容稳定
# 正文长度连续 3 次采样不变 → 内容已完整加载

# 6. 提取内容
result = await page.evaluate("""
    () => {
        const body = document.querySelector('.markdown-body, article, main, [role="main"]');
        return {
            url: location.href,
            title: document.querySelector('h1, .doc-title')?.textContent?.trim() || document.title,
            text: body?.textContent?.trim() || '',
        };
    }
""")
```

## 异常处理

| 现象 | 检查顺序 |
|------|----------|
| 正文一直不出现 | 1.是否需要登录 2.selector 是否选错 3.正文是否在 iframe 4.是否需要先点击目录项 |
| 页面一直加载 | 用 `commit` 代替 `networkidle`，等正文文本 |
| 只有骨架屏 | JS 错误？反爬限制？需要特定语言/地区路径？ |
| 内容与 URL 不匹配 | 核对 `page.url()`、h1、面包屑 |

## 读取图片

```python
images = await page.evaluate("""
    () => [...(document.querySelector('.markdown-body')?.querySelectorAll('img') || [])]
        .map(img => ({
            src: img.currentSrc || img.src || img.getAttribute('data-src') || '',
            alt: img.alt || '',
            complete: img.complete,
            width: img.naturalWidth,
            height: img.naturalHeight,
        }))
""")
```

判断图片真实可用：`complete: true` 且 `width > 0` 且 `height > 0`。

## 虚拟列表页面

表现：DOM 文本长度不随滚动增长，上方内容被卸载。

策略：每滚动一段就读取 → 用标题/文本片段去重 → 持续累计 → 无新增时停止。
