# 文档抓取流程

合并自：[[文档抓取方法论]]、[[华为开发者文档抓取实战规范]]、[[华为开发者文档拉取经验总结]]

---

## Phase 0: 范围确认

开始前确认 4 件事：

1. **抓什么范围** — 哪些 catalog、哪些模块
2. **怎么分类** — 本地目录映射关系
3. **最终目录长什么样** — 先建目录，再抓内容
4. **landing page 要不要** — HTML 归档保留，Markdown 产出不保留

如果跳过这一步，最常见的问题是：抓多了、抓偏了、分类不一致、产物需要返工。

## Phase 1: 获取导航目录

### 1.1 优先使用 getCatalogTree API

```python
resp = session.post(
    "https://svc-drcn.developer.huawei.com/community/servlet/consumer/cn/documentPortal/getCatalogTree",
    json={"language":"cn","catalogName":"harmonyos-guides"}
)
tree = resp.json()["value"]["catalogTreeList"]
```

### 1.2 提取叶子节点

递归遍历 `catalogTreeList` → `children`，收集所有 `isLeaf: true` 且含 `relateDocument` 的节点。`relateDocument` 即为文档 slug。

### 1.3 目录树清洗

官网导航树中可能插入冗余中间层级（如"开发"、"指南"）。构建本地路径时需过滤无意义的中间节点名。

### 1.4 用户分类与官网分类的映射

官网一级节点不一定对应用户期望的分类目录。抓取前先输出完整节点列表，让用户确认映射关系。

## Phase 2: 差异对比

### 2.1 批量获取官方更新时间

```python
def fetch_doc(catalog, slug):
    resp = session.post(API_URL, json={
        "objectId": slug, "version": "",
        "catalogName": catalog, "language": "cn"
    })
    val = resp.json()["value"]
    return val["title"], val["displayUpdateTime"], val["content"]["content"]
```

对所有已收录文档的 slug 调用此 API，获取 `displayUpdateTime`，与本地 Markdown 中的 `更新时间` 对比。不一致则标记为需更新。

### 2.2 新增文档检测

将 Phase 1 获取的全量叶子节点 slug 与本地 manifest 条目做差集：
- `official_slugs - local_slugs` = 新增文档（需拉取）
- `local_slugs - official_slugs` = 已下线文档（需标记）

## Phase 3: 拉取正文

### 3.1 单文档拉取

直接使用 `getDocumentById` API（不需要 Playwright）。返回的 `content.content` 即为完整 HTML 正文。

### 3.2 图片处理

见 [[03-图片资源规范]]。核心原则：
- 优先复用本地已有 assets（图片内容极少随文档更新而改变）
- 新图片按 `file-{timestamp}-{hash}.{ext}` 命名存入 `assets/{docName}/`

### 3.3 断点续传

千级以上文档必须设计检查点：
- 每批次完成后即时写入 manifest
- 重跑时读取已有 manifest 跳过已完成项

## Phase 4: HTML → Markdown

详见 [[02-HTML转Markdown规范]]。

## Phase 5: 后处理

转换完成后必须对全量 Markdown 执行自动化后处理：

1. **清除 `[h2]/[h3]/[h4]` 标记** — API 返回的标题文本含这些前缀
2. **图片引用独立成行** — 与正文文本拆分
3. **行首空格清除** — 段落前导 2+ 空格去除（保护代码块、列表、表格、引用）
4. **段落内连续空格压缩** — 4+ 空格压缩为单空格
5. **API 元数据间距** — references 文档的 `**xxx：**` 行之间插入空行

## Phase 6: 质量检查

### 检查清单

- [ ] 各目录文件数是否合理
- [ ] manifest 条目数与文件数是否对应
- [ ] 是否存在 404 页面混入
- [ ] 更新时间是否更新
- [ ] 表格是否有分隔线（`|---|---|`）
- [ ] 代码块是否有缩进
- [ ] 是否残留 `[h2]/[h3]/[h4]` 标记
- [ ] 图片引用是否独立成行
- [ ] API 文档是否有 `支持设备：` 元数据

## manifest 规范

```json
{
  "group": "分类路径",
  "slug": "文档slug",
  "name": "文档标题",
  "file": "相对路径/文件名.md",
  "url": "https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/slug",
  "title": "文档标题"
}
```

字段说明：
- `group`：目录分组路径
- `slug`：官网 URL 尾部标识
- `file`：本地文件相对路径
- `url`：官网来源完整 URL

## 异常处理

### 异常 slug
URL 命名不符合规律但页面有效 → 按同组规则修正本地文件名，manifest 中同步更新。

### 404 页面
标题为 `404` 或正文非目标内容 → 不保留，从 manifest 剔除。

### API 返回 document not found (code: 92531031)
说明 catalogName 不正确或文档已下线 → 检查 catalog 映射表。

## 清理原则

任务完成后保留：
- 最终 Markdown 文件
- `manifest.json`
- assets 目录

任务完成后删除：
- 临时脚本
- 调试输出
- 中间 HTML 文件
- 缺失 URL 列表
