## 目的

这篇文档只讨论一件事：

- 面对动态加载网页时，如何实时打开、等待、滚动、观察和阅读页面内容

它不覆盖后续归档流程；这些内容已经由其他文档负责。本文的重点是“当前任务中如何可靠读到页面真实内容”。

## 一、适用场景

实时阅读动态网页适合以下情况：

- 页面首屏只有壳，正文由前端 JS 后续注入
- 页面内容依赖滚动触发加载
- 图片、代码块、表格或示例需要滚动后才出现
- 目录树、标签页、折叠块会影响正文可见性
- 页面有虚拟列表，DOM 只保留当前可视区域
- 需要快速理解页面内容，但不需要离线保存

典型对象包括：

- 官方文档站
- API 参考页
- 前端框架渲染的指南页
- 长列表搜索结果页
- 带目录联动的技术文章
- 需要点击展开的组件文档

## 二、核心原则

实时阅读的目标不是“页面加载事件完成”，而是“目标正文已经可读”。

不要只依赖这些信号：

- `domcontentloaded`
- `load`
- `networkidle`
- 页面标题出现
- 外层容器存在

更可靠的判断是：

- 正文标题已经出现
- 正文容器存在
- 正文文本长度达到合理阈值
- 关键章节标题已经出现
- 滚动后正文长度不再明显变化
- 需要阅读的图片、表格、代码块已经进入 DOM

核心思路：

1. 用真实浏览器打开页面
2. 等正文真实出现
3. 按阅读路径滚动和展开内容
4. 持续观察正文是否还在变化
5. 只读取当前任务需要的信息

## 三、推荐工具

主工具：

- `Playwright`

辅助能力：

- 浏览器截图：确认页面是否真的渲染
- DOM 查询：确认正文、标题、图片、代码块是否出现
- 网络观察：判断内容是否来自异步请求
- 控制台日志：排查前端渲染错误

`curl` 只适合做辅助探测，例如：

- 检查 URL 是否可访问
- 检查重定向
- 检查状态码
- 观察初始 HTML 是否包含正文

对于动态正文，主路径仍应使用浏览器。

## 四、实时阅读标准流程

### 1. 打开页面

建议使用较早的进入条件，然后自己判断正文是否出现：

```ts
await page.goto(url, {
    waitUntil: 'commit',
    timeout: 45000,
});
```

原因：

- 一些动态页面长期保持网络连接
- `networkidle` 可能永远不稳定
- `load` 完成不代表正文已经注入
- 真正重要的是正文是否可读

### 2. 等待正文可读

通用等待逻辑：

```ts
async function waitForReadableContent(page) {
    await page.waitForFunction(() => {
        const title = document.querySelector('h1, .doc-title')?.textContent?.trim() || '';
        const body = document.querySelector('.markdown-body, article, main, [role="main"]');
        const text = body?.textContent?.trim() || '';

        return title.length > 0 && text.length > 200;
    }, { timeout: 45000 });
}
```

这个阈值不能机械套用，要按页面类型调整：

- 短 API 页面可以降低文本长度阈值
- 长指南页面应等待更多章节出现
- 搜索结果页应等待结果列表项出现
- 组件文档应等待示例区域出现

### 3. 确认正文容器

优先寻找语义正文区域：

1. 站点明确正文容器，例如 `.markdown-body`
2. `article`
3. `main`
4. `[role="main"]`
5. 人工确认的内容容器

不要把整页 `body` 当正文读取，除非页面非常简单。

整页 `body` 通常会混入：

- 顶部导航
- 侧边目录
- 页脚
- 登录入口
- 分享按钮
- 语言切换
- 推荐文章

### 4. 分段滚动

很多页面只有滚动后才会加载完整内容。

建议分段滚动，而不是一次跳到底：

```ts
async function autoScroll(page) {
    await page.evaluate(async () => {
        await new Promise<void>((resolve) => {
            let total = 0;
            const distance = 600;
            const timer = window.setInterval(() => {
                window.scrollBy(0, distance);
                total += distance;

                if (total >= document.body.scrollHeight - window.innerHeight) {
                    window.clearInterval(timer);
                    resolve();
                }
            }, 300);
        });
    });
}
```

滚动时要观察：

- 页面高度是否增长
- 正文文本是否增长
- 图片是否从占位变成真实内容
- 代码块是否渲染出来
- 目录高亮是否随正文变化

### 5. 展开正文内容

一些内容需要点击后才可读：

- 展开
- 更多
- 查看全部
- Show more
- Expand
- 代码示例标签页
- API 参数标签页

只在正文区域内点击这些控件。

不要误点这些会改变阅读范围的控件：

- 下一篇
- 返回首页
- 登录
- 下载 SDK
- 反馈
- 站点导航菜单

### 6. 等待内容稳定

滚动和展开后，需要确认正文不再继续变化。

可以用正文长度作为简单稳定条件：

```ts
async function waitForStableText(page) {
    let lastLength = 0;
    let stableCount = 0;

    while (stableCount < 3) {
        const length = await page.evaluate(() => {
            const body = document.querySelector('.markdown-body, article, main, [role="main"]');
            return body?.textContent?.trim().length || 0;
        });

        if (length === lastLength) {
            stableCount += 1;
        } else {
            stableCount = 0;
            lastLength = length;
        }

        await new Promise((resolve) => setTimeout(resolve, 800));
    }
}
```

也可以结合结构信号判断：

- 章节标题数量不再增加
- 图片数量不再增加
- 代码块数量不再增加
- 表格数量不再增加
- 页面高度不再增加

## 五、实时阅读图片

本文不讨论图片下载和附件保存，只讨论阅读时如何确认图片真实可见。

动态页面中的图片常见问题：

- 初始 `src` 是占位图
- 真实地址在 `data-src`
- 需要滚动到附近才加载
- 使用 `<picture>` 和 `srcset`
- 图片加载失败但 alt 文本存在

阅读时应确认：

- `img.currentSrc` 是否有值
- 图片是否完成加载
- 图片自然尺寸是否大于 0
- 图片是否在正文区域内
- alt 文本是否能补充理解

可用检查：

```ts
const images = await page.evaluate(() => {
    const body = document.querySelector('.markdown-body, article, main, [role="main"]');
    return [...(body?.querySelectorAll('img') || [])].map((img) => ({
        src: img.currentSrc || img.src || img.getAttribute('data-src') || '',
        alt: img.alt || '',
        complete: img.complete,
        width: img.naturalWidth,
        height: img.naturalHeight,
    }));
});
```

判断标准：

- `complete: true`
- `width > 0`
- `height > 0`

如果图片对理解页面很重要，但没有加载出来，应继续滚动到图片附近，或用截图确认页面实际状态。

## 六、实时阅读代码块

代码块经常由前端组件二次渲染。

阅读时要确认：

- 代码块内容已经出现
- 代码没有被行号污染
- 代码没有混入复制按钮文字
- 当前标签页是否是目标语言
- 示例是否需要点击其他语言标签

常见噪声：

- 复制
- Copy
- 自动换行
- 深色主题
- 收起
- 展开

读取代码时优先取 `pre` 或 `code` 的真实文本。

```ts
const codeBlocks = await page.evaluate(() => {
    const body = document.querySelector('.markdown-body, article, main, [role="main"]');
    return [...(body?.querySelectorAll('pre, code') || [])]
        .map((node) => node.textContent?.trim() || '')
        .filter(Boolean);
});
```

如果页面存在语言标签页，应先确认当前显示的是任务需要的语言，例如：

- ArkTS
- TypeScript
- JavaScript
- Kotlin
- Swift
- C++

## 七、实时阅读表格

动态文档里的表格可能有横向滚动、折叠列或虚拟渲染。

阅读时要确认：

- 表头已经出现
- 所有关键列可见
- 横向滚动区域没有隐藏重要字段
- 展开按钮没有折叠后半部分
- 表格不是只渲染了可视区域

如果表格很宽，不能只读屏幕上看到的一部分。应检查 DOM 中的完整单元格文本。

```ts
const tables = await page.evaluate(() => {
    const body = document.querySelector('.markdown-body, article, main, [role="main"]');
    return [...(body?.querySelectorAll('table') || [])].map((table) =>
        [...table.querySelectorAll('tr')].map((row) =>
            [...row.querySelectorAll('th, td')].map((cell) =>
                cell.textContent?.trim() || ''
            )
        )
    );
});
```

## 八、目录和锚点辅助阅读

很多长文档有侧边目录或页内锚点。

实时阅读时可以用目录辅助判断：

- 当前页面有哪些章节
- 正文是否缺少某些章节
- 滚动是否已经到达底部
- 点击目录项后是否切换了正文

但不要只读取目录文本当正文。

目录通常只提供结构，真正内容仍然要从正文容器中读取。

## 九、处理虚拟列表页面

有些页面为了性能，会在滚动时卸载已经离开视口的 DOM。

表现：

- 页面看起来内容很多
- DOM 文本长度却没有持续增长
- 向下滚动后上方内容从 DOM 中消失
- 列表项节点数量基本固定

这种页面不能等到最后再一次性读取 DOM。

实时阅读策略：

1. 每滚动一段就读取当前可见内容
2. 用标题、链接或文本片段去重
3. 持续累计已读内容
4. 滚动到无新增内容时停止

判断“无新增”时不要只看页面高度，应看新增条目的数量。

## 十、处理 iframe

有些文档正文、示例或交互预览在 iframe 中。

如果主页面找不到正文，应检查：

- 是否存在 iframe
- iframe 是否同源
- iframe 内是否有正文容器
- 示例内容是否需要在 iframe 内读取

Playwright 中可以枚举 frame：

```ts
for (const frame of page.frames()) {
    const title = await frame.title().catch(() => '');
    const text = await frame.locator('body').innerText().catch(() => '');
    console.log(title, text.slice(0, 200));
}
```

如果 iframe 跨域且不能直接读取 DOM，可以使用截图或页面可见文本作为辅助判断。

## 十一、异常处理

### 1. 正文一直不出现

检查顺序：

1. 是否需要登录
2. 是否被 Cookie、地区、语言弹窗挡住
3. selector 是否选错
4. 正文是否在 iframe
5. 是否需要先点击目录项
6. 是否需要展开折叠块
7. 页面是否实际返回 404 或空状态

### 2. 页面一直加载不停

不要无限等 `networkidle`。

处理方式：

- 改用 `commit`
- 等正文文本
- 等关键章节
- 用滚动和稳定条件判断可读性

### 3. 页面只显示骨架屏

检查：

- 网络请求是否失败
- 控制台是否有 JS 错误
- 是否被反爬或权限限制
- 是否需要特定语言或地区路径

### 4. 内容和 URL 不匹配

动态站点可能路由跳转后仍保留旧壳。

需要核对：

- `page.url()`
- 页面 `h1`
- 面包屑
- 正文第一段
- 目录高亮项

如果这些信息不一致，说明页面可能没有正确切换到目标文档。

## 十二、实时阅读最小伪代码

```ts
const browser = await chromium.launch();
const page = await browser.newPage();

await page.goto(url, { waitUntil: 'commit', timeout: 45000 });
await waitForReadableContent(page);
await autoScroll(page);
await expandContentControls(page);
await waitForStableText(page);

const result = await page.evaluate(() => {
    const body = document.querySelector('.markdown-body, article, main, [role="main"]');
    const title = document.querySelector('h1, .doc-title')?.textContent?.trim() || document.title;

    return {
        url: location.href,
        title,
        text: body?.textContent?.trim() || '',
        headings: [...(body?.querySelectorAll('h1, h2, h3') || [])].map((h) =>
            h.textContent?.trim() || ''
        ),
        imageCount: body?.querySelectorAll('img').length || 0,
        codeBlockCount: body?.querySelectorAll('pre, code').length || 0,
        tableCount: body?.querySelectorAll('table').length || 0,
    };
});

await browser.close();
```

这个结果用于当场阅读、判断和回答问题，不负责落盘归档。

## 十三、实时阅读检查清单

阅读动态网页时，至少确认：

1. URL 是否是目标页面
2. 标题是否与目标文档一致
3. 正文容器是否选对
4. 正文文本是否达到合理长度
5. 关键章节是否已经出现
6. 页面是否滚动到底或达到目标章节
7. 折叠内容是否已经展开
8. 图片是否真实加载，而不是占位图
9. 代码块是否真实出现，且没有混入控件文字
10. 表格是否完整，不只是可视区域
11. iframe 中是否还有正文或示例
12. 页面是否存在 404、登录墙、空状态或地区限制
13. 内容是否还在继续动态变化

## 十四、最终结论

实时阅读动态加载网页的关键是：

- 不等“网页加载完”，而是等“正文可读”
- 不信单一加载事件，而是看正文、章节、图片、代码和表格是否真实出现
- 不一次性跳到底，而是按阅读路径滚动、展开和观察
- 不把导航、目录、页脚当正文
- 不把保存归档流程混进实时阅读流程

后续归档流程应继续参考已有规范文档。
